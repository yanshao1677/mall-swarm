# 阶段 1：延迟消息发送

## 本项目落地状态

| 维度 | 状态 |
|------|------|
| 代码位置 | `sendDelayMessageCancelOrder` + `CancelOrderSender` |
| 默认是否启用 | ✅ `generateOrder` 成功后会调用 |
| 依赖 | RabbitMQ 可用、`oms_order_setting` 有数据 |

## 触发条件

- `generateOrder` 事务成功：订单已 insert、库存已 lock、购物车已删
- 调用 `sendDelayMessageCancelOrder(orderId)`（`generateOrder` 末尾 L245）

## 输入字段

| 字段 | 类型 | 来源 |
|------|------|------|
| orderId | Long | 新生成订单主键 |
| normalOrderOvertime | Integer | `oms_order_setting.id=1` 字段 `normal_order_overtime`（分钟） |

## 判定规则

```
delayTimes = normalOrderOvertime * 60 * 1000   // 毫秒
exchange   = "mall.order.direct.ttl"
routeKey   = "mall.order.cancel.ttl"
body       = orderId (Long)
expiration = String.valueOf(delayTimes)  // per-message TTL
```

SQL 种子数据：`normal_order_overtime = 120`（分钟）→ delayTimes = 7_200_000 ms。

## 状态读写位置

- 读：MySQL `oms_order_setting`
- 写：RabbitMQ TTL 队列 `mall.order.cancel.ttl`（消息带 expiration）

## 正常路径

1. `orderSettingMapper.selectByPrimaryKey(1L)`
2. 计算 delayTimes
3. `cancelOrderSender.sendMessage(orderId, delayTimes)`
4. `amqpTemplate.convertAndSend(exchange, routeKey, orderId, MessagePostProcessor 设置 expiration)`
5. 日志 `send orderId:{}`

## 分支路径

- `POST /order/cancelOrder` Controller **也调用** sendDelayMessageCancelOrder（重发延迟消息，非立即取消）→ 详见 [陷阱-cancelOrder接口名与行为不一致](陷阱-cancelOrder接口名与行为不一致.md)

## 失败处理

- MQ 不可用 → 发送抛异常；订单已提交则**无自动取消**（除非定时扫表）
- 证据不足：generateOrder 与 sendMessage 是否同一事务外——send 在 @Transactional 方法末尾，事务提交后发消息

## 幂等性 / 一致性约束

- 同一 orderId 多次发送 → 多条延迟消息，阶段 2 幂等消化
- expiration 为 per-message，非队列级固定 TTL

## 代码骨架

```java
public void sendDelayMessageCancelOrder(Long orderId) {
    OmsOrderSetting setting = orderSettingMapper.selectByPrimaryKey(1L);
    long delayTimes = setting.getNormalOrderOvertime() * 60 * 1000;
    cancelOrderSender.sendMessage(orderId, delayTimes);
}

public void sendMessage(Long orderId, final long delayTimes) {
    amqpTemplate.convertAndSend(
        QueueEnum.QUEUE_TTL_ORDER_CANCEL.getExchange(),
        QueueEnum.QUEUE_TTL_ORDER_CANCEL.getRouteKey(),
        orderId,
        message -> {
            message.getMessageProperties().setExpiration(String.valueOf(delayTimes));
            return message;
        });
}
```

## 最小验证清单

- 下单后 MQ 中出现带 expiration 的消息
- normalOrderOvertime=120 时 expiration=7200000
- TTL 到期后消息出现在 `mall.order.cancel` 队列
