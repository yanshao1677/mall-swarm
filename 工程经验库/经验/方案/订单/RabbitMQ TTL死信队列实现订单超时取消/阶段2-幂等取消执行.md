# 阶段 2：幂等取消执行

## 本项目落地状态

| 维度 | 状态 |
|------|------|
| 代码位置 | `CancelOrderReceiver` + `cancelOrder` |
| 默认是否启用 | ✅ MQ 消费链路 |

## 触发条件

- `CancelOrderReceiver` 从队列 `mall.order.cancel` 收到消息体 `orderId`（Long）
- 或手动 `POST /order/cancelUserOrder` 直接调用 `cancelOrder(orderId)`

## 输入字段

| 字段 | 类型 | 来源 |
|------|------|------|
| orderId | Long | MQ 消息体 |

## 判定规则

**查询条件（必须同时满足才取消）**：

```
id = orderId
status = 0        // 待付款
deleteStatus = 0
```

```
IF cancelOrderList 为空:
    RETURN  // 静默，无异常
ELSE:
    order.status = 4   // 已关闭
    releaseSkuStockLock(orderItemList)  // lock_stock 减
    updateCouponStatus(couponId, memberId, 0)  // 恢复未使用
    IF useIntegration != null:
        返还积分给会员
```

## 状态读写位置

- 读/写：MySQL `oms_order`、`oms_order_item`、`pms_sku_stock`
- 写：优惠券状态、会员积分

## 正常路径

1. `OmsOrderExample` 按三条件查询
2. 空列表 → return
3. 更新 status=4
4. 查 orderItemList
5. releaseSkuStockLock
6. 恢复优惠券、积分

## 分支路径

- 已支付 status=1 → 查不到 → **静默 return**（MQ 迟到不误关）
- 已关闭 status=4 → 同上
- 重复消费同一 orderId → 第二次查不到 status=0 → 静默

## 失败处理

- DB 更新失败 → 抛异常，MQ 可重试；若已改 status 则重试幂等

## 幂等性 / 一致性约束

- **近似严格幂等**：以 status=0 为闸门，重复消息不改变已支付单
- 不保证 exactly-once 副作用（优惠券/积分返还仅首次成功执行）

## 代码骨架

```java
public void cancelOrder(Long orderId) {
    OmsOrderExample ex = new OmsOrderExample();
    ex.createCriteria()
        .andIdEqualTo(orderId)
        .andStatusEqualTo(0)
        .andDeleteStatusEqualTo(0);
    List<OmsOrder> list = orderMapper.selectByExample(ex);
    if (CollectionUtils.isEmpty(list)) {
        return;
    }
    OmsOrder order = list.get(0);
    order.setStatus(4);
    orderMapper.updateByPrimaryKeySelective(order);
    // ... release stock, coupon, integration
}
```

## 最小验证清单

- status=0 订单取消后变 4，lock_stock 释放
- status=1 订单调用 cancelOrder 无变更
- 同一 orderId 连续调用两次，第二次无 DB 写
