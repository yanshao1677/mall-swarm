# 业务断言失败经全局异常转响应

## 本项目落地状态

| 维度 | 状态 |
|------|------|
| 代码位置 | `mall-common/Asserts` + `GlobalExceptionHandler` |
| 默认是否启用 | ✅（admin/portal/auth 等 `scanBasePackages=com.macro.mall` 可扫到） |

## 1. 问题

Service 层业务校验失败（库存不足、用户禁用）需统一转 API 响应，避免 Controller 层层 if-return。

## 2. 核心思路

`Asserts.fail(message)` 抛 `ApiException`；`@RestControllerAdvice` 捕获 → `CommonResult.failed(message)` code=500。参数校验 `MethodArgumentNotValidException`/`BindException` → 取首个 FieldError → `validateFailed` code=404。

## 3. 注意

业务错误与系统错误均为 code=500（ApiException），与 validateFailed 404 区分。

## 4. 测试点

- Service 中 Asserts.fail 后客户端收到 failed message
- @Valid 失败返回 404 与字段提示
