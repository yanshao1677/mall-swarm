# 跨服务认证契约常量集中定义

## 本项目落地状态

| 维度 | 状态 |
|------|------|
| 代码位置 | `mall-common/AuthConstant` |
| 默认是否启用 | ✅ gateway、admin、auth、portal 均引用 |

## 1. 问题

网关、登录聚合服务、admin、portal 需约定相同 clientId、Redis key、Session 键、Header 名，分散硬编码易不一致。

## 2. 核心思路

共享 jar 中 `AuthConstant` 接口集中定义：

| 常量 | 值 |
|------|-----|
| ADMIN_CLIENT_ID | admin-app |
| PORTAL_CLIENT_ID | portal-app |
| PATH_RESOURCE_MAP | auth:pathResourceMap |
| JWT_TOKEN_HEADER | Authorization |
| JWT_TOKEN_PREFIX | Bearer  |
| STP_ADMIN_INFO | adminInfo |
| STP_MEMBER_INFO | memberInfo |

## 3. 使用方

- AuthController：clientId 分支
- UmsResourceServiceImpl：Redis Hash 名
- SaTokenConfig：PATH_RESOURCE_MAP 读取
- StpInterfaceImpl / 登录 Service：Session 键
