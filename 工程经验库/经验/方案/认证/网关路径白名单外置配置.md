# 网关路径白名单外置配置

## 本项目落地状态

| 维度 | 状态 |
|------|------|
| 代码位置 | `mall-gateway/application.yml` → `IgnoreUrlsConfig` |
| 默认是否启用 | ✅ |

## 1. 问题

登录、注册、静态文档、支付回调等路径不能走鉴权过滤器，且列表会随业务迭代变更，不宜硬编码在 Java 中。

## 2. 核心思路

YAML 配置 `secure.ignore.urls` 列表，通过 `@ConfigurationProperties(prefix="secure.ignore")` 绑定到 `IgnoreUrlsConfig.urls`，注入 `SaReactorFilter.setExcludeList(urls)`。

## 3. 处理流程

请求进入过滤器 → 路径命中 excludeList → 跳过全部 setAuth 逻辑 → 直接路由。

## 4. 典型白名单条目类型

- API 文档：`/doc.html`、`/*/v3/api-docs/**`
- 监控：`/actuator/**`
- 整模块公开：`/auth/**`、`/search/**`
- 登录注册：`/portal/sso/login`、`/admin/admin/login`
- 第三方回调：`/portal/alipay/**`

## 5. 测试点

- 白名单路径无 Authorization 可访问
- 非白名单无 token 返回 401
- 新增公开 API 后改 YAML 即可，无需改过滤器代码
