# 鉴权异常统一映射 401/403 响应

## 本项目落地状态

| 维度 | 状态 |
|------|------|
| 代码位置 | `mall-gateway/SaTokenConfig.handleException` |
| 默认是否启用 | ✅ |

## 1. 问题

网关鉴权失败若返回框架默认 HTML/空体，前端与移动端无法统一处理；需与业务 API 相同 JSON 契约。

## 2. 核心思路

`SaReactorFilter.setError` 自定义处理：`NotLoginException` → `CommonResult.unauthorized()`（code=401）；`NotPermissionException` → `CommonResult.forbidden()`（code=403）；并设置 `Content-Type: application/json` 与 CORS 头。

## 3. 具体实现

```java
private CommonResult handleException(Throwable e) {
    SaHolder.getResponse()
        .setHeader("Content-Type", "application/json; charset=utf-8")
        .setHeader("Access-Control-Allow-Origin", "*");
    if (e instanceof NotLoginException) return CommonResult.unauthorized(null);
    if (e instanceof NotPermissionException) return CommonResult.forbidden(null);
    return CommonResult.failed(e.getMessage());
}
```

## 4. 测试点

- 无 token 访问受保护路径 → HTTP body `code:401`
- 有 token 无权限 → `code:403`
- 响应 Content-Type 为 application/json
